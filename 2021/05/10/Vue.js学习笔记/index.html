VUE.js学习笔记

1. 简介

vue和react的相同点

  • 利用虚拟DOM实现快速渲染
  • 轻量级
  • 响应式组件
  • 支持服务器端渲染
  • 易于继承路由工具，打包工具以及状态管理工具

什么是虚拟DOM

传统的web开发，是利用jQuery操作DOM，这是非常消耗资源的

我们可以在JS的内存中构建类似于DOM的对象，去拼装数据，拼装完整后，把数据整体解析，一次性插入到HTML中，就形成了虚拟DOM

VUE1.0没有虚拟DOM，VUE2.0改成了基于虚拟DOM

VUE是一个渐进式框架

渐进式框架意味着你可以将VUE作为应用的一部分嵌入其中，带来更丰富的交互体验

或者如果你希望将更多的业务逻辑使用Vue实现，那么Vue的核心库以及其生态系统。比如Core+Vue-router+Vuex，也可以满足你各种各样的需求。

VUE的生命周期

生命周期函数

  • beforeCreate：创建实例之前执行的钩子函数，此时数据和方法还没有绑定到应用对象app上
  • created：创建实例之后执行的钩子函数，此时数据和方法绑定到应用对象app上
  • beforeMount：将编译完成的HTML挂载到对应虚拟dom时触发的钩子，此时页面并没有内容，渲染之前，根据数据生成的DOM对象是获取不到的。
  • mounted：编译好的HTML挂载到页面完成后执行的事件钩子，此钩子函数中一般会做一些请求ajax获取数据进行数据初始化，此时可以获取DOM对象
  • beforeUpdate：更新之前的钩子，修改之前会触发
  • updated：更新之后的钩子
  • beforeDestory：实例销毁之前执行的钩子
  • destroyed：实例销毁完成执行的钩子

生命周期总结

  1. 首先是创建一个Vue实例，也就是执行var vm = new Vue()

  2. 然后是初始化一些生命周期函数和默认的事件，其它的东西都还没创建，此时data和methods还没有初始化

  3. 在第二步执行完之后会执行**beforeCreate()**钩子函数，但是在该函数中是拿不到data和methods的数据

  4. 加载data和methods

  5. 在上一步执行完之后，调用钩子函数**created()**，在该函数中可以可以调用methods中的方法和访问data中的数据

  6. 判断有没有el属性，有的话就将el属性所对应的编译成模板template,把Vue代码中的那些指令进行执行，最终在内存中生成一个编译好的最终模板字符串，然后将这个模板字符串渲染为内存中的DOM，此时，只是 在内存中渲染好了模板，并没有把模板挂载到真正的页面中去。

  7. 上一步执行完之后，调用钩子函数beforeMounted，此时模板已经在内存中编译完成，但是还没有把模板渲染到页面中，

    这时候页面中的元素还没有内真正替换过来，只是之前写的一些模板字符串。例如在该函数中输出document.getElementById('name').innerHTML，输出的内容只是{{message}}，还没有被真正替换

  8. 将内存中编译好的模板真实的替换到浏览器的页面中去

  9. 上一步执行完之后，调用钩子函数mounted，表示内存中的模板已经挂载到了页面中，用户已经可以看到渲染好的页面了。该函数是实例创建期间最后一个生命周期函数，当执行完mounted就表示，实例已经被完全创建好了。

  10. 1-9都是组件创建阶段，之后就进入了组件运行阶段

    组件运行阶段的生命周期函数只有两个，beforeUpdated()和updated()

    这两个事件会根据data数据的改变，有选择的触发一次或多次

    调用beforeUpdated的时候数据已经被更新，但是界面还没有被更新

    执行完beforeUpdated会先根据data中最新的数据在内存中重新渲染出一份最新的DOM树，当最新的内存DOM树被更新后，会把最新的内存DOM树重新渲染到真实的页面中，这时候就完成了数据从data到view的更新

    执行updated的时候数据和界面都已经被更新，它们保持一致

  11. 当执行beforeDestoryed的时候就已经从运行阶段进入到了销毁阶段，当执行beforeDestory的时候，实例上面的所有的data和methods以及其它数据都处于可用状态，还没有真正执行销毁的过程

  12. 当执行destoryed的时候所有的数据已经进入到了不可用的状态，此时组件已经被完全销毁了

Vue对象的选项

1). el

el是挂载点，会管理el选项命中的元素以及内部的后代元素
同时也可以使用其他的的选择器，但是建议使用id选择器
除了可以设置div，也可以设置其他的dom元素，但不是使用html和body

2). data

对象或函数类型
指定初始化状态属性数据的对象
vm也会自动拥有data中所有属性
页面中可以直接访问使用
数据代理: 由vm对象来代理对data中所有属性的操作(读/写)

3). methods

包含多个方法的对象
供页面中的事件指令来绑定回调
回调函数默认有event参数, 但也可以指定自己的参数
所有的方法由vue对象来调用, 访问data中的属性直接使用this.xxx

4). computed

包含多个方法的对象
对状态属性进行计算返回一个新的数据, 供页面获取显示
一般情况下是相当于是一个只读的属性
利用set/get方法来实现属性数据的计算读取, 同时监视属性数据的变化
如何给对象定义get/set属性
    在创建对象时指定: get name () {return xxx} / set name (value) {}
      对象创建之后指定: Object.defineProperty(obj, age, {get(){}, set(value){}})

5). watch

包含多个属性监视的对象
分为一般监视和深度监视
    xxx: function(value){}
    xxx : {
        deep : true,
        handler : fun(value)
    }
另一种添加监视方式: vm.$watch('xxx', function(value){})

6. VUE指令

指的是以v-开头的一组特殊语法

v-text

设置标签的文本值

v-html

设置元素的innerHTML，内容中右html结构会被解析为标签，而v-text指令不论内容是什么都只解析为文本。

解析文本使用v-text,需要解析html结构使用v-html

v-pre

将代码原封不动的解析出来，不做任何处理。

v-pre用于跳过这个元素和它子元素的编译过程，用于显示原本的Mustache语法

v-on

为元素绑定事件

事件名不需要写on

指令可以简写为@

绑定的方法定义在methods属性中

方法内部通过this关键字可以访问定义在data中的数据

计数器

v-show

v-show指令的作用是：根据真假切换元素的显示状态

原理是修改元素的display，实现显示隐藏

指令后面的内容，最终都会解析为布尔值

值为true元素显示，值为false元素隐藏

数据改变之后，对应元素的显示状态会同步更新

v-if和v-else

根据表达式的真假切换元素的显示状态

本质是通过操纵dom元素来切换显示状态

表达式为true，元素存在于dom树中，为false，从dom树中删除

v-if和v-else中间不能有元素，v-if为true的时候v-else不显示，v-if为false时v-else会显示，多种情况可以使用v-else-if。

不止可以写boolean变量，还可以写一个简单的判断

v-bind

为元素绑定属性

完整写法是v-bind：属性名

简写的话可以直接省略v-bind，只保留：属性名

需要动态的增删class建议使用对象的方式

：key可以保证不同块的唯一性，防止VUE偷懒，不重新渲染某些input元素

  • 数组语法

一次性插入，不再修改

图片切换

v-for

  • 根据数据生成列表结构

  • 数组经常和v-for结合使用

  • 语法是(item,index) in 数据

  • item和index可以结合其它指令一起使用

  • 数组长度的更新会同步到页面上，是响应式的

v-on

事件绑定的方法写成函数调用的方式，可以传入自定义参数

定义方法时需要定义形参来接收传入的实参

事件的后面跟上.修饰符可以对事件进行限制

.enter可以限制触发的按键为回车

如果需要同时传入某个参数，同时需要event，可以通过$event传入参数

VUE提供了一些修饰符来帮助我们方便的处理一些事件

.stop - 调用 event.stopPropagation()。
.prevent - 调用 event.preventDefault()。
.{keyCode | keyAlias} - 只当事件是从特定键触发时才触发回调。
.native - 监听组件根元素的原生事件。
.once - 只触发一次回调。

v-model

获取和设置表单元素的值（双向数据绑定）

绑定的数据会和表单元素值相关联

绑定的数据<–>表单元素的值，无论修改谁都会改变

修饰符

  • lazy修饰符：

    • 默认情况下，v-model默认是在input事件中同步输入框的数据的。

      也就是说，一旦有数据发生改变对应的data中的数据就会自动发生改变。lazy修饰符可以让数据在失去焦点或者回车时才会更新

  • number修饰符

    默认情况下，在输入框中无论我们输入的是字母还是数字都会被当作字符串类型进行处理

    但是如果我们希望处理的是数字类型，那么最好直接将内容当作数字处理

  • trim修饰符

    如果输入的内容收尾有很多空格，通常我们希望将其去除

    trim修饰符可以过滤内容左右两边的空格

7. 计算属性和监视

计算属性

  1. 在computed属性对象中定义属性的方法
  2. 在页面中使用来显示计算的结果

监视属性

  1. 通过vm对象的$watch()或者watch配置来监视指定指定的属性。
  2. 当属性变化时，回调单数自动调用，在函数内部进行计算

计算属性高级

  1. 通过getter/setter实现对属性数据的显示和监视
  2. 计算属性存在缓存，多次读取只执行一次getter计算

getter和setter的理解

vue在组件本身是一个对象，当将此对象传入vue的实例的时候，vue会对data进行处理，通过JavaScript的对象方法：object.defineProperty (对象属性定义)将data每一个属性进行getter和setter化，

这样，当组件实例化完成的时候，data之下的所有属性已经被处理了，所以可以被vue内部追踪到每个属性的变化

并且，每个组件的实例化都会有一个相应的watch实例对象，它会在组件渲染的过程中，它会在组件渲染的过程中把属性记录为依赖，之后当依赖项的 setter 被调用时，会通知 watcher 重新计算，从而致使它关联的组件得以更新，对dom新型重新渲染

总结：其它组件发生时，会调用依赖这些组件的set方法，

当组件发生变化时，也会调用get方法，
