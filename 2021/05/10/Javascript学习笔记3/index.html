4.DOM

document object model,文档对象模型。

节点

构成HTML网页的最近本单元，网页中的每一部分都可以称之为是一个节点，比如html标签，属性，文本，知识，整个文档都是一个节点。

虽然都是节点，但是实际上他们的具体类型是不同的。常见节点分为四类：

  • 文档节点（文档）：整个 HTML 文档。整个 HTML 文档就是一个文档节点。
  • 元素节点（标签）：HTML标签。
  • 属性节点（属性）：元素的属性。
  • 文本节点（文本）：HTML标签中的文本内容（包括标签之间的空格、换行）。

节点的类型不同，属性和方法也都不尽相同。所有的节点都是Object。

什么是DOM

DOM：Document Object Model，文档对象模型。DOM 为文档提供了结构化表示，并定义了如何通过脚本来访问文档结构。目的其实就是为了能让js操作html元素而制定的一个规范。

DOM就是由节点组成的。

解析过程： HTML加载完毕，渲染引擎会在内存中把HTML文档，生成一个DOM树，getElementById是获取内中DOM上的元素节点。然后操作的时候修改的是该元素的属性。

事件的冒泡

解释

事件的冒泡指的就是事件的向上传导，当后代元素上的事件被触发时，其祖先元素的相同事件也会被触发。

在开发中，大多数情况冒泡都是有用的。如果不希望发生事件冒泡可以通过事件对象来取消冒泡。

取消冒泡

事件的委派

指将事件统一绑定给元素共同的祖先元素，这样当后代元素上的事件触发时，会一直冒泡到祖先元素。

事件委派是利用了冒泡，通过委派可以减少事件绑定的次数，提高程序的性能。

事件的绑定

使用对象.事件=函数的形式绑定响应函数，它只能同时为一个元素的一个事件绑定响应函数，不能绑定多个，如果绑定了多个，则后面的会覆盖前面的。该方法不支持IE8及以下浏览器，在ie8中可以使用attachEvent()来绑定事件。但是它是后绑定的先执行

addEventListener()

可以同时为一个元素的相同事件同时绑定多个响应函数。这样当事件被触发时，响应函数会按照函数的绑定顺序执行。

参数

  1. 事件的字符串，不要on
  2. 回调函数，当事件触发时该函数会被调用
  3. 是否在捕获阶段触发事件，需要一个布尔值，一般都传false

事件的传播

事件的三个阶段

  1. 捕获阶段

    1. 在捕获阶段时从最外层的祖先元素向目标元素进行事件的捕获，但是默认此时不会触发事件

  2. 目标阶段

    事件捕获到目标元素，捕获结束开始在目标元素触发事件

  3. 冒泡阶段

    事件从目标元素向它的祖先元素传递，依次触发祖先元素的事件

如果希望在捕获阶段就触发事件，可以将addEventListener()的第三个参数设置为true

一般情况下我们不会希望在捕获阶段就触发事件，所以这个参数一般都是false

IE8及以下没有捕获阶段。

闭包

1.产生的条件

当一个嵌套的内部函数引用了嵌套的外部函数的变量就产生了闭包

2.闭包的理解

理解一：闭包是嵌套的北部函数

理解二：包含被引用变量的的对象

注意：闭包存在于嵌套的内部函数

执行函数定义就会产生闭包

不会产生闭包

3.常见的闭包

  1. 将函数作为另一个函数的返回值
  2. 将函数作为实参传递给另一个函数调用

4.闭包的理解

函数f1的返回值是位于它内部的函数f2，所以var result=f1()这句是将函数f2赋值给了全局变量result。

函数每次被调用的时候都会传进去一个环境，也就是this，result是一个全局变量，所以执行result()往里面传的this是window，我验证了一下

结果：

因为函数f2引用了f1中的变量，所以在执行var result=f1()之后函数f1并没有从内存中销毁，而函数f2当中没有n这个变量，就会往上一层找，f2的上一层就是f1，所以弹出的n就是f1中n的值。

nAdd()同理，就是相当于将f1中的一个匿名函数赋值给了全局变量nAdd，也是因为闭包的原因，所以可以对f1中的n进行++的操作，再次执行result()就会弹出1000。

我对代码进行了一下修改，f2没有使用f1中的变量，f2的存在并不依赖f1，也就没有产生闭包，再执行完var result=f1();之后f1也随之销毁。
